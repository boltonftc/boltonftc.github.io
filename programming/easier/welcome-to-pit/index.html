Welcome to the PIT

You are looking at the PIT Training Station -- your personal workspace for learning how to program an FTC robot. Before you write any code, let's take a quick tour so you know where everything is and how it works.

What You See On Screen

Your screen is split into three areas:

Top-Left -- LESSONS (you are reading this right now!) This panel walks you through each lesson step by step. You will read instructions here and follow along in the code editor.

Bottom-Left -- BENCH ROBOT and DRIVER HUB The bench robot is a 3D preview of your robot sitting on a workbench -- just like a real pit table at a competition. You can click and drag to orbit the camera around it.

Next to it is the Driver Hub. In real FTC, the Driver Hub is the Android tablet that drivers use to control the robot. Here it shows your robot's status, lets you pick which program to run, and displays telemetry messages from your code.

Right Side -- CODE EDITOR This is where you write Java code that controls the robot. It works like a simple text editor with line numbers and syntax coloring. You will type code here, save it, and compile it to see your changes on the robot.

What Is a Code Package?

A code package is your personal folder of Java files. Think of it as your own copy of the robot's brain. Every student gets their own package so you can experiment without affecting anyone else's code.

When you create a package, you start with a clean template that has the basic robot structure already set up. As you complete lessons, you will add code to this template and the robot will learn new abilities -- driving, turning, using sensors, and more.

Create Your Code Package

The Edit-Save-Compile-Run Loop

Every time you change code, you follow the same four steps:

1. Edit -- Type or change code in the editor. 2. Save -- Click the Save button above the editor. It turns orange when you have unsaved changes. 3. Compile -- Click Recompile on the Driver Hub. This translates your Java code into instructions the robot can understand. If there is a mistake, you will see an error message instead. 4. Run -- Select your program from the TeleOp dropdown, then click INIT followed by START.

You will do this loop dozens of times. It becomes second nature.

The Driver Hub Up Close

The Driver Hub has several important controls:

  • TeleOp / Auto dropdowns -- Pick which program to run.

For now you will only use TeleOp.

  • INIT button -- The big circle in the center. This loads

your program onto the robot and gets it ready.

  • START -- After INIT, tap the center button again to start

your program running.

  • Recompile button -- Compiles your saved code. You must

do this after every edit before running.

  • CODE button -- Switch between code packages.
  • Console area -- The bottom section shows telemetry

messages from your running program.

What About the Robot?

Right now, your robot does not drive. That is on purpose!

The starter code has the basic structure -- it knows about its motors and how to read the gamepad -- but there are no driving instructions yet. As you work through the lessons, you will add code piece by piece:

  • Early lessons teach you to send messages to the Driver

Hub and wire up the joystick so the robot can drive.

  • Later lessons add turning, strafing, sensors, and more.

By the end, you will have a fully drivable robot that you built from scratch -- one lesson at a time.

Navigating Lessons

Each lesson has colored blocks to help you know what to do:

  • Gold blocks are key ideas. Write these in your

engineering notebook.

  • Green blocks contain code you need to type into the

editor. Copy it carefully!

  • Blue blocks are action steps -- things to do right now.
  • Purple blocks are hints and extra explanation.
  • CHALLENGE blocks are optional extra-credit activities

for when you want to go deeper.

Some lessons have a clickable link that jumps the code editor to the right spot so you know exactly where to type.

At the top of a lesson you will find:

  • Reset to Start -- Reverts your code back to the starting

point for this lesson. Use this if you get stuck or made a mistake you cannot fix.

At the bottom of every lesson you will find:

  • Mark Complete -- Tells the system you are done with this

lesson so you can move to the next one.

Can I Edit Code Outside This Editor?

Yes! Your code package is just a folder of .java files on your computer. You can open them in Visual Studio Code or any text editor. Just remember to click Recompile on the Driver Hub after saving changes externally.

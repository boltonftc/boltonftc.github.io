Your Code Package

The Robot's Brain — Written by You

In a previous lesson, you learned what the parts of an FTC robot are — motors, sensors, the Control Hub, and so on. But all of that hardware is useless without code. The code is the robot's brain. It decides when to spin a motor, what to do when a button is pressed, and how to navigate the field during autonomous mode.

When you created your code package, the system gave you a set of Java files — your personal copy of the robot's brain. Java is a programming language, similar to Python or JavaScript, that is used to tell computers exactly what to do step by step. It is the official language for FTC robot programming.

Think of your code package like a notebook that already has the chapter headings printed in. The structure is there — the files, the sections, the comments that mark where code belongs — but the pages are blank. Your job over the next several lessons is to fill them in, one chapter at a time.

Before we start writing real code, let's take a tour of those files so you know what each one is for and how they fit together.

💡 Key Idea
Your **code package** is a set of Java files that control the

robot. Each file has a specific job — driving, button actions, sensors, navigation. You will add code to these files as you complete lessons.

The Two Halves of Every Robot Program

Before we go any further, there is one concept you need to understand about how robot programs are structured. Every FTC program has two halves:

1. Initialization (Before the Match)

The first half runs before the driver presses START. This is the init phase. During init, your code:

  • Connects to hardware (motors, sensors, cameras)
  • Sets up starting values (which direction motors spin, what

units sensors use)

  • Shows a "Ready" message on the Driver Hub
  • Basically says: *"I am preparing everything. Do not move

yet."*

In MecanumDrive.java, this is everything above the waitForStart() line. That line is the dividing point — it pauses your program and waits for the driver to press START.

2. The Operational Loop (During the Match)

The second half runs after START is pressed. This is the loop — a while (opModeIsActive()) block that repeats over and over, roughly 60 times per second, for the entire match.

Each time through the loop, your code:

  • Reads the gamepad inputs (which buttons are pressed, where

the joystick is)

  • Reads sensor values (heading, position, distance)
  • Decides what to do (how fast to spin each motor)
  • Sends commands to the hardware
  • Updates the telemetry display

When the match ends (or the driver presses STOP), the loop exits and the program shuts down.

💡 Key Idea
Every FTC program has two phases:

Init (before START) — set up hardware, configure settings. Loop (after START) — read inputs, make decisions, control motors. The loop repeats ~60 times per second until STOP.

Your First Code Change — Leave Your Mark

You have toured the files and understand the structure. Now let's make your first edit — something small to prove that the whole save-and-compile cycle works. Nothing flashy. Just a comment.

A comment in Java is a line that starts with //. The computer completely ignores it — comments are notes for humans only. Programmers use comments to explain what their code does, leave reminders, or mark sections.

You are going to add a comment with your name at the top of MecanumDrive.java. It will not change how the robot behaves at all — but it will prove you can edit code, save it, and compile it without errors. That is the foundation for everything that comes next.

Save and Compile

Now let's confirm the computer accepts your change.

✅ To Do
1. Click **Save** in the Code Editor (or press Ctrl+S).
  1. On the Driver Hub, click Recompile.
  2. Wait for the compile to finish. You should see a green

"BUILD SUCCESSFUL" message.

If you see a red error instead, double-check that you started your comment with // and that you did not accidentally delete any of the surrounding code. Comments are forgiving — as long as the line starts with //, Java will ignore everything after it.

💜 Hint
**What if I get an error?** Errors usually mean you accidentally

changed a line of real code. Look at the error message — it will give you a line number. Go to that line and make sure it matches the original. Your comment line starting with // should not cause any errors.

Congratulations — you just made your first code change, saved, and compiled. That is the same cycle you will use for every lesson going forward:

  1. Read the lesson instructions
  2. Type code in the editor
  3. Save and Recompile
  4. Test on the robot

The cool stuff is coming soon -- you will make text appear on the Driver Hub screen. But every program you ever write on this robot begins the same way: edit, save, compile, test.

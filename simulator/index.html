The FTC 3D Simulator (PIT Mode) is a custom simulator that lets students write real FTC Java code and test it on a virtual robot — before touching any physical hardware. Code written in the sim runs unmodified on a real robot. The May 2026 release adds a second robot and a second lesson track to support both the 2025–26 and 2026–27 seasons.

Two Robots, Two Lesson Tracks

Both robots on the field at the same time — they can practice against each other.

The simulator now ships with two distinct robots and two matching lesson tracks:

  • Shooter Track — uses the original shooter bot with intake, flywheel, and indexer servo. Follows the structure of the 2025–26 Decode season. This is the default track and has the most complete lesson set.
  • Dumper Track — introduces a new dumper bot that intakes up to 3 balls and uses a servo-controlled dump bed to deliver them to the 5× high goal. Aligned with the 2026–27 BioBuzz season.

Both robots can be used on the same practice field and can run against each other — useful for testing defense, spacing, and cycle timing.

Switching to the Dumper Track

If you've already been working through the Shooter track and have completed lessons up through MecanumDrive, here's the recommended path to get the dumper bot up and running quickly:

  1. Create a new code package and select the Dumper track at the prompt.
  2. Open the SparkFun OTOS lesson. This sets up the optical odometry sensor used for field-centric drive on the dumper bot. SparkFun OTOS lesson.
  3. Use Reset Lesson Code to apply the full MecanumDrive field-centric code to your package in one step. This gives you a driving robot immediately — no need to re-type the drive code. Reset Lesson Code pulls in the complete, tested drive code.
  4. Complete the Intake and Servo lessons to enable the dumper bot's intake roller and dump bed. Intake and Servo lessons add the two remaining mechanisms.
What's intentionally skipped: This path does not go through the OTOS autonomous, AprilTag, or Pedro Pathing lessons. Those add significant complexity and aren't needed to get a competitive dumper bot on the field. We'll revisit autonomous for the dumper later.

This code path has been tested on a real robot. A few small adjustments may be needed when transferring to physical hardware:

  • Intake motor power direction may need to be inverted depending on how the motor is mounted
  • Servo home and dump positions will need to be tuned to match your specific mechanism geometry

Exporting Code to Your Real Robot

When you're ready to move from the simulator to physical hardware, use Export Code Package. This creates a zip file containing all the Java files in your package.

Export Code Package — generates a zip of your Java files.

Open the zip and copy the files — all of them, or just the ones you need — into your Android project's ./teamcode folder. The code is standard FTC SDK Java and compiles with no modifications.

The simulator and the real robot use the same SDK interfaces (DcMotor, Servo, IMU, etc.). There are no simulator-specific imports to remove — what you write in PIT Mode is already real robot code.

--- layout: page title: "Bolton FTC Laptop Setup (Windows 11)" ---

This guide prepares a Windows 11 laptop for FTC programming and robot testing. The goal is a consistent setup across all team laptops so students can switch machines without losing time.

About FIRST Tech Challenge (FTC): FTC is a robotics program where students design, build, wire, and code robots for a new game each season. Teams practice engineering process skills: testing, iteration, debugging, and communicating design decisions.

1. Core Software Install

  1. Install Android Studio: developer.android.com/studio.
  2. Follow Brogan Pratt's Android Studio + FTC SDK setup video: YouTube setup guide.
  3. If you already have your team Git repository, clone that repo instead of creating a fresh SDK project.
  4. Install the REV Hardware Client: REV install guide.
  5. Install the Impulse 3D Simulator from boltonftc.github.io/simulator/.

2. FTC Gradle Dependencies

Add or verify these entries in build.dependencies.gradle:

repositories { mavenCentral() google() // Needed for androidx maven { url 'https://jitpack.io'} maven { url = 'https://maven.brott.dev/' } } dependencies { implementation 'org.firstinspires.ftc:Inspection:11.1.0' implementation 'org.firstinspires.ftc:Blocks:11.1.0' implementation 'org.firstinspires.ftc:RobotCore:11.1.0' implementation 'org.firstinspires.ftc:RobotServer:11.1.0' implementation 'org.firstinspires.ftc:OnBotJava:11.1.0' implementation 'org.firstinspires.ftc:Hardware:11.1.0' implementation 'org.firstinspires.ftc:FtcCommon:11.1.0' implementation 'org.firstinspires.ftc:Vision:11.1.0' implementation 'androidx.appcompat:appcompat:1.2.0' }

Add or verify these entries in TeamCode/build.gradle:

// Include common definitions from above. apply from: '../build.common.gradle' apply from: '../build.dependencies.gradle' android { namespace = 'org.firstinspires.ftc.teamcode' packagingOptions { jniLibs.useLegacyPackaging true } } dependencies { implementation project(':FtcRobotController') implementation 'com.github.Pedro-Pathing.PedroPathing:core:v2.0.6' implementation 'com.acmerobotics.dashboard:dashboard:0.6.0' }

3. USB Wi-Fi + Network Reliability Setup

  1. Plug in the USB Wi-Fi adapter.
  2. Open Network Connections and rename adapters so one is clearly robot-facing (for example: FTC ROBOT).
  3. Verify which adapter reliably connects to the robot network. Not all adapters behave the same.
  4. In Windows Wi-Fi settings, forget all existing SSIDs except your active robot network.
  5. Keep your other adapter focused on internet Wi-Fi only, so Windows does not jump robot/internet networks unexpectedly.

4. Folder + Desktop Shortcut Setup

  1. Create C:\ftc and put a desktop shortcut to it.
  2. In Android Studio, right-click TeamCode and choose Open In > Explorer.
  3. Create a desktop shortcut to the TeamCode folder that contains the Java files.

6. Git Branch Per Laptop

Each laptop should use its own branch to avoid accidental conflicts.

git checkout -b laptop1 git status

Use branch names like laptop1, laptop2, etc. Run git status often so students confirm which branch they are on before coding.

7. FTC Git + SSH Setup (Dual-Remote Pattern)

This is the workflow we use so we can pull updates from FTC SDK while pushing team code to our own repository.

  1. Start from an FTC SDK-based project (or your existing team codebase with FTC structure).
  2. Create a base snapshot commit:
    git add . git commit -m "Initial FTC 2026 team codebase"
  3. Check current remotes:
    git remote -v
  4. Create SSH key on the laptop:
    ssh-keygen -t ed25519 -C "ftc-laptop-1"
  5. Copy and add public key to GitHub account (Settings > SSH and GPG keys).
  6. Test auth:
    ssh -T git@github.com
  7. Rename FTC SDK remote to upstream:
    git remote rename origin upstream
  8. Add team repository as origin:
    git remote add origin git@github.com:boltonftc/team2_code_2026.git
  9. Verify:
    git remote -v
  10. Push base code to team repo:
    git push -u origin master

Final layout: upstream = official FTC SDK, origin = Bolton FTC team repository.

8. Cloning to a Second Laptop

  1. Generate a unique SSH key on the second laptop:
    ssh-keygen -t ed25519 -C "ftc-laptop-2"
  2. Add that laptop's public key to GitHub (Settings > SSH and GPG keys).
  3. Test SSH access:
    ssh -T git@github.com
  4. Clone the team repo to C:\ftc:
    git clone git@github.com:boltonftc/team2_code_2026.git
  5. Open the project in Android Studio.
  6. Create and switch to a laptop-specific branch:
    git checkout -b laptop2

9. Common Student Git Commands
